Críticos

6 problemas — Bugs e funcionalidades quebradas

Preloader não funciona

Crítico · index.js:64
O preloader usa a classe animate__animated animate__fadeOut (do Animate.css), mas a biblioteca não está carregada no projeto. O preloader simplesmente desaparece abruptamente após 200ms, sem animação de saída.
Adicionar CDN do Animate.css ou substituir por transição CSS nativa em modern.css.

Código morto: Jarallax

Crítico · index.js:37
A linha home.setAttribute('data-jarallax-original-styles', ...) referencia a biblioteca Jarallax, que não está carregada em nenhum lugar do projeto. Código morto de uma versão anterior.
Remover a linha 37 de index.js.

helpers.js nunca é carregado

Crítico · assets/js/helpers.js
O arquivo helpers.js não é referenciado em nenhum <script> no HTML. Além disso, ele duplica as funções select(), selectAll() e isVariableDefined() já existentes em index.js, e contém um bug lógico na função isVariableDefined (typeof !!el sempre retorna "boolean").
Deletar o arquivo helpers.js e unificar as funções em index.js.

index.ts é código morto

Crítico · index.ts
O arquivo apenas executa console.log("Hello via Bun!"). Não serve ao site e nunca é referenciado pelo HTML ou pelo servidor.
Deletar index.ts.

Conflito .btn-primary (verde vs laranja)

Crítico · style.css:3417 + modern.css:209
O Bootstrap define .btn-primary com background-color: #4cbd89 (verde), mas o modern.css redefine com background: var(--coral-orange) (laranja, #ff5a45). Isso cria uma inconsistência grave de identidade visual. O mesmo ocorre com .btn-dark e .btn base.
Definir uma paleta única e remover as duplicatas de style.css ou usar PurgeCSS para eliminar o Bootstrap não utilizado.

55+ chamadas externas para avatares

Crítico · index.html:479-841
Todos os cards de alunos usam a URL ui-avatars.com para gerar avatares via API. São mais de 55 requisições externas. Se o serviço ficar fora do ar, todos os avatares dos alunos serão perdidos.
Substituir por placeholders locais, gerar os avatares via CSS com as iniciais, ou armazenar as imagens localmente.

Altos & Médios

10 problemas — SEO, acessibilidade, performance e segurança

Sem meta tags Open Graph / Twitter Card

Alto · index.html:head
Faltam tags og:title, og:description, og:image, og:url e twitter:card. O site não exibe preview quando compartilhado em redes sociais.
Adicionar as meta tags OG e Twitter Card no <head>.

style.css com 21.497 linhas não minificado

Alto · assets/css/style.css
Bootstrap 5.1.0 completo, não minificado e com muitos componentes não utilizados (tabelas, formulários, validação, tooltips, popovers, etc.). O projeto usa apenas grid, navbar, collapse, modal e utilities básicas.
Usar PurgeCSS para extrair apenas os estilos utilizados, ou fazer um build customizado do Bootstrap.

Nenhuma imagem usa lazy loading

Alto · index.html (todas as imagens)
Nenhuma das 60+ imagens do site usa o atributo loading="lazy". Todas carregam eager, impactando o tempo de carregamento inicial.
Adicionar loading="lazy" a todas as imagens abaixo da dobra.

Falta de atributos ARIA e acessibilidade

Médio · index.html (vários locais)
O canvas de ondas (<canvas>) não tem aria-label. Os links sociais do time têm apenas "in", "ig", "gh" como texto, sem aria-label. Os emojis decorativos (&#9997;&#65039;, &#129309;) não têm aria-hidden="true".
Adicionar aria-label, aria-hidden="true" e melhorar textos dos links sociais.

Back-to-top usa <a href="#"> em vez de <button>

Médio · index.html:730
O botão "voltar ao topo" é um link (<a>) com href="#". Semanticamente deveria ser um <button>.
Substituir por <button> com aria-label="Voltar ao topo".

Canvas animation roda mesmo fora da viewport

Médio · index.js:72-110 (waveCanvas)
A função waveCanvas() usa requestAnimationFrame sem verificar se o canvas está visível. O loop de animação continua rodando mesmo quando o usuário rola para outra seção.
Usar IntersectionObserver para pausar a animação quando fora da viewport.

PDF em iframe é frágil e pesado

Médio · index.html:923
O PDF de Classificados é carregado em um <iframe>. Muitos navegadores fazem download do PDF ou exibem área em branco em vez de renderizar inline. Além disso, é pesado para carregar.
Usar PDF.js para renderização inline segura, ou abrir em nova aba com target="_blank".

Arquivo com espaço e caractere especial no nome

Médio · assets/images/Símbolo.logo (2).png
O arquivo Simbolo.logo (2).png contém espaços, parênteses e caractere acentuado. Isso pode causar problemas em servidores Linux e em URLs.
Renomear para simbolo-logo.png (sem espaços, sem acentos).

serve.ts sem headers de segurança e compressão

Médio · serve.ts
O servidor Bun não implementa headers de segurança (CSP, X-Frame-Options, CORS), nem compressão gzip/brotli. Também não trata corretamente arquivos vazios.
Adicionar middlewares de segurança e compressão ao servidor.

apresentacao_front.html sem suporte a touch/swipe

Médio · apresentacao_front.html
A apresentação de slides não suporta navegação por toque (touch/swipe) em dispositivos móveis, e o CSS inline impede scroll, podendo travar usuários de teclado.
Adicionar suporte touch/swipe e garantir navegação por teclado.

Baixos — Manutenibilidade

10 problemas — Qualidade de código e boas práticas

~60+ declarações !important no CSS

Baixo · modern.css (seção "AJUSTES FINOS")
O arquivo modern.css contém aproximadamente 60 declarações !important, especialmente na seção "AJUSTES FINOS" (linhas 1424-1916). Isso indica uma guerra de especificidade e dificulta a manutenção.
Refatorar o CSS para usar especificidade adequada em vez de !important.

.hero-card definido 3 vezes em modern.css

Baixo · modern.css:353, 1485, 1965
A classe .hero-card é definida três vezes ao longo do arquivo, cada vez com !important para sobrepor as anteriores. O mesmo ocorre com .glass-nav.
Consolidar todas as definições em único local por componente.

Classes CSS não utilizadas

Baixo · modern.css
As classes .hero-mini-card, .classes-showcase, .hero-actions, .cta-panel e .light-icon estão definidas no CSS mas não são usadas em nenhum elemento do HTML.
Remover as classes não utilizadas do CSS.

package.json sem scripts de start/build

Baixo · package.json
O package.json não possui uma seção "scripts". Não há comandos definidos para iniciar o servidor, build ou desenvolvimento.
Adicionar "start": "bun run serve.ts" e "dev": "bun --watch run serve.ts".

README.md genérico do Bun

Baixo · README.md
O README contém apenas o boilerplate padrão do Bun. Não documenta o projeto real, sua estrutura, dependências ou como usar.
Escrever um README específico do projeto com instruções reais.

@types/bun usa versão "latest"

Baixo · package.json
A dependência @types/bun: "latest" é frágil. Pode quebrar o projeto a qualquer momento com uma atualização.
Fixar a versão exata (ex: "@types/bun": "^1.2.0").

Extensões de imagem misturadas (.jpg e .jpeg)

Baixo · assets/images/
Na mesma pasta há arquivos .jpg e .jpeg. Isso é inconsistente e pode causar links quebrados se a extensão errada for referenciada.
Padronizar todas as imagens para .jpg (ou todas para .jpeg).

CSS e JS inline em apresentacao_front.html

Baixo · apresentacao_front.html (529 linhas CSS + 30 JS)
O arquivo de apresentação tem 529 linhas de CSS e 30 linhas de JS tudo inline. Não reutiliza os estilos do projeto.
Externalizar CSS e JS em arquivos próprios e referenciar os estilos do projeto.

coolvetica.css vs modern.css: .logo-font duplicado

Baixo · coolvetica.css + modern.css:133
A classe .logo-font está definida tanto em coolvetica.css (linha 46) quanto em modern.css (linha 133). A versão do modern.css é mais robusta (usa fallback var(--display)).
Manter apenas a definição do modern.css e remover a duplicada.

Falta de structured data (JSON-LD)

Baixo · index.html:head
Não há dados estruturados JSON-LD para a organização ou programa social. Isso ajuda mecanismos de busca a entenderem o conteúdo.
Adicionar JSON-LD com nome, descrição, logo e URL do programa.
